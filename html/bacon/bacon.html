Passo a Passo: Algoritmo para o Problema dos Seis Graus de Bacon

O problema dos seis graus de Bacon consiste em encontrar o caminho mais curto entre dois atores em uma rede de filmes, onde os nós são os nomes dos atores e as arestas representam filmes em que os atores apareceram juntos.

Algoritmo de Busca em Largura (BFS)

Inicialização:

  • Comece com uma fila vazia (aqui chamada de queue).
  • Adicione o nó inicial (ator inicial) à fila.

Exploração:

  • Enquanto a fila não estiver vazia:
    • Remova o primeiro nó da fila.
    • Se este nó for o nó de destino (ator de destino), retorne o caminho até ele.
    • Caso contrário, marque o nó como visitado.
    • Para cada vizinho não visitado do nó atual:
      • Crie um novo caminho que inclua este vizinho.
      • Adicione este novo caminho à fila.

Condição de Parada:

  • Se a fila estiver vazia e o nó de destino não for encontrado, retorne que não há caminho.

Resultado:

  • Quando encontrar o nó de destino, retorne o caminho até ele e o comprimento do caminho.

Este algoritmo é aplicado para encontrar o caminho mais curto entre dois atores no grafo de atores (actorsGraph), onde os nós são os nomes dos atores e as arestas representam filmes em que os atores apareceram juntos. O BFS é eficaz para encontrar o caminho mais curto em grafos não ponderados, como é o caso aqui.
